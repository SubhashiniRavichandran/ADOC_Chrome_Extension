# Extension Icons

This directory should contain the following icon files:

## Required Icons

1. **icon16.png** (16x16 pixels)
   - Used in the extension toolbar
   - Favicon for extension pages

2. **icon48.png** (48x48 pixels)
   - Used in the Chrome Extensions management page
   - Extension details page

3. **icon128.png** (128x128 pixels)
   - Used in the Chrome Web Store
   - Installation dialog
   - Extension management page (large view)

## Design Guidelines

### Style
- Modern, clean design
- Recognizable at small sizes
- Consistent with ADOC branding
- Use brand colors: Purple gradient (#667eea to #764ba2)

### Format
- PNG format with transparency
- RGB color mode
- Optimized file size

### Suggested Design
The icon should represent:
- Data quality/reliability (checkmark, shield, or quality badge)
- Integration/connection (link, bridge, or connection symbol)
- ADOC branding (use brand colors and style)

### Example Icon Concept
```
┌─────────────┐
│    ╔══╗     │  - Purple gradient background
│    ║  ║     │  - White checkmark or data symbol
│    ╚══╝     │  - Rounded corners
└─────────────┘
```

## Creating Icons

### Tools
- Adobe Illustrator
- Figma
- Sketch
- Online icon generators (e.g., favicon.io)

### Steps
1. Design the icon in vector format at the largest size (128x128)
2. Export as PNG with transparency
3. Scale down to 48x48 and 16x16
4. Optimize each size for clarity (minor adjustments may be needed)
5. Test at all sizes in Chrome

### Testing Icons
1. Load the extension in Chrome
2. Check toolbar icon (16x16)
3. Check extensions page (48x48 and 128x128)
4. Verify clarity and visibility at all sizes
